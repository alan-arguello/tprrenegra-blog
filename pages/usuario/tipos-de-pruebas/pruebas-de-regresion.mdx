# Pruebas de regresión manuales

Las pruebas de regresión verifican que tu producto siga funcionando correctamente incluso después de ser modificado (mejoras de software, parches, cambios de configuración, etc.) o cuando el entorno que lo rodea cambia (nuevas versiones de navegadores, actualizaciones de servidores, etc.).

Durante las pruebas de regresión, se pueden descubrir nuevos errores de software. Una de las principales razones para realizar pruebas de regresión es determinar si un cambio en una parte del software afecta a otras partes.

Las pruebas de regresión se pueden realizar para probar un sistema de manera eficiente mediante la selección sistemática del conjunto mínimo apropiado de pruebas necesarias para cubrir adecuadamente un cambio en particular.

## Cómo realizar pruebas de regresión manuales

<Steps>

### Paso 1
Sigue un guión. Los guiones pueden venir en forma de casos de uso, casos de prueba e incluso wireframes.

### Paso 2
Realiza las pruebas en todos los navegadores y plataformas que tengan una cuota de mercado significativa.

</Steps>

<Callout>
  Las pruebas de regresión también deben ser realizadas por los desarrolladores de software (o sus contrapartes de control de calidad) a medida que cambian el producto. El hecho de que los gerentes de producto o los investigadores de UX realicen estas pruebas no significa que los desarrolladores de software estén excluidos de realizarlas también.
</Callout>

Para obtener más información sobre las pruebas de regresión, consulta [este artículo de Wikipedia](https://es.wikipedia.org/wiki/Pruebas_de_regresi%C3%B3n).
